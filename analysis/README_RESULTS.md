# Producing the numbers for the paper

The paper's Results section (Dataset Overview, RQ1, RQ2, RQ3) is populated
from a single script:

    analysis/run_all_analyses.py

It imports the same helpers the four notebooks use (`analysis/common.py`) —
with `deduplicate=True` — so the numbers are directly comparable to the
figures those notebooks produce.

## Requirements

* Python 3.10+
* `pandas`, `pyarrow` (or `fastparquet`), `seaborn`, `matplotlib`
* The Pass B output tables under `data/nature_of_agent_conflicts/`:
  `universe.parquet`, `internal_merges.parquet`,
  `classified_chunks.parquet`, `resolved_chunks.parquet`,
  `extraction_errors.parquet`.
* *(Optional, for raw-AIDev counts)* the raw AIDev parquet drop, searched
  for in `data/raw/aidev/`, `data/aidev/`, or `data/`. The script skips
  that section gracefully if neither location exists.

## Running

From the project root:

    python -m analysis.run_all_analyses

or:

    python analysis/run_all_analyses.py

## Outputs

Two files are written to `analysis/results/`:

* `results_summary.json` — full-precision, machine-readable dump of every
  aggregate the paper cites. This is the canonical artefact.
* `results_summary.txt` — the same content pretty-printed for eyeballing.

## What the script reports

1. **Dataset characterisation.** Headline counts (projects, PRs, internal
   merges, conflicting merges, conflict chunks) before and after
   merge-level deduplication on `(repo_full_name, merge_sha)`; per-agent,
   per-language, per-task-type breakdowns; PR fan-out distribution (how
   many PRs reference each physical merge).
2. **RQ1 — Structural nature.** Descriptive stats for chunks-per-merge
   (all merges and conflicting-only), LOC stats for V1/V2/base/resolution,
   localization success rate, PR outcome breakdown, and conflict
   incidence per stratum.
3. **RQ2 — Resolution strategies.** Strategy distribution with and
   without the `Imprecise` bucket (paper uses the "without" view and
   reports the Imprecise share separately); per-stratum breakdowns for
   agent, language, `pr_task_type`, and `resolver_type`.
4. **RQ3 — Resolver.** Agent-vs-human resolver counts at the merge level
   and strategy-by-resolver breakdown at the chunk level.

## Feeding the numbers back into the paper

Once `results_summary.json` exists, the TODO markers in `paper/main.tex`
(Dataset Overview and the RQ1 / RQ2 / RQ3 Results subsections) can be
filled in by referring to the following keys:

* `characterisation.headline` → Dataset Overview headline numbers.
* `characterisation.internal_merges.pr_fanout_*` → the deduplication
  paragraph in the PR Selection section.
* `rq1.chunks_per_merge_*_desc`, `rq1.*_loc_desc`,
  `rq1.conflict_incidence` → RQ1 Results text and tables.
* `rq2.global_excl_imprecise`, `rq2.imprecise_share_global`,
  `rq2.per_stratum` → RQ2 Results text.
* `rq3.resolver_counts_merge_level`,
  `rq3.strategy_by_resolver_excl_imprecise` → RQ3 Results text.

The figures themselves continue to be generated by the notebooks
(`rq1_structural_nature.ipynb`, `rq2_resolution_strategies.ipynb`,
`rq3_resolver.ipynb`).
